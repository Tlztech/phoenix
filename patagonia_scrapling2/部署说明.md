# Windows 部署说明（监视任务 + 自动断点续跑）

本方案在一台 Windows 10 电脑上部署爬虫，并用「任务计划」每 10 分钟监视一次：
**程序被 Akamai 限流会自动停止 → 监视任务等 30 分钟 → 自动断点续跑 → 重复直到全部抓完。** 全程无人值守。

> **程序是否自带"被限流就停止"？** —— **是的，已内置。**
> 抓取时若连续被拦（默认 18 次）判定为限流，程序会**自动结束并保留断点**，退出码：
> `0`=全部完成、`1`=被限流/未完成、`2`=连类别页都被挡。监视脚本据此决定"等冷却后续跑"还是"已完成停止"。

---

## 一、原理（三个部件）

1. **爬虫 `main.py`**：抓取；被限流自动退出并保留断点（`output\.checkpoint_*.jsonl`）；再次运行会自动跳过已抓的、继续抓剩下的；全部完成后写正式文件并删除断点。
2. **监视脚本 `windows-deploy\monitor.ps1`**：每次被任务计划触发时检查——
   - 程序在跑 → 不动；
   - 程序没跑、断点在、距上次活动 ≥30 分钟 → **启动续跑**；
   - 程序没跑、断点在、还没到 30 分钟 → **继续等**（冷却）；
   - 没有断点但已有成品文件 → **判定全部完成，停止**；
   - 首次（没断点没成品）→ **启动**。
3. **任务计划**：每 10 分钟触发一次 `monitor.ps1`。

这样就实现了你要的：**限流即停 → 冷却 30 分钟 → 自动续跑 → 循环到取完。**

---

## 二、前置条件

1. **Python 3.10+** —— <https://www.python.org/downloads/>（安装勾选 "Add python.exe to PATH"）。验证：`python --version`。
2. **Google Chrome** —— <https://www.google.com/chrome/>（爬虫驱动它过 Akamai，必须装）。
3. **拷贝整个工程目录**到目标电脑（比如 `C:\patagonia_scraper`）。`output\`、`.chrome-profile\` 会自动生成，不用拷。

---

## 三、部署步骤

### 1. 装依赖
在工程目录打开 PowerShell（地址栏输入 `powershell` 回车），执行：
```powershell
python -m pip install -r requirements.txt
```
> 目标机若没有 Google Chrome 又装不了，可执行 `python -m playwright install chromium`，并在 monitor.ps1 里给 main.py 加 `--chrome-path` 指向它；否则优先用 Chrome。

### 2. 配置要爬的类别
编辑 `input_url.txt`（每行一个类别 URL，`#` 开头是注释）。默认：
```
https://www.patagonia.jp/shop/womens
https://www.patagonia.jp/shop/mens
https://www.patagonia.jp/shop/kids-baby/big-kids
```

### 3. （如需要）改 monitor.ps1 里的 Python 路径
`windows-deploy\monitor.ps1` 顶部：
- 若 `python` 不在 PATH，把 `$Python = "python"` 改成完整路径，如 `"C:\Python312\python.exe"`。
- 冷却分钟数默认 `$CooldownMin = 30`，一般不用改。

### 4. 安装并启动监视任务
右键 `windows-deploy\install_task.ps1` →「使用 PowerShell 运行」；或在工程目录执行：
```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\windows-deploy\install_task.ps1
```
它会依次：
1. **把上次的成品**（`patagonia_output_*.xlsx` 和 `_summary.txt`）**移到 `output\history\`**——避免监视把旧成品误判成"本轮已完成"；
2. 创建每 10 分钟触发的任务 `PatagoniaScraperMonitor`；
3. **立即触发一次开始抓取**。

> 安全保护：若检测到还有**断点**（说明上一轮没跑完/在进行中），install 不会开新一轮，而是提示"监视任务会自动续跑那一轮"。想强制全新重跑，先手动删掉 `output\.checkpoint_*` 再运行 install。

---

## 四、查看进度 / 停止

**看监视日志**（工程根目录）：
```powershell
Get-Content .\monitor.log -Wait
```
日志是英文（脚本为跨代码页兼容用纯 ASCII），对照：
- `running, skip` = 抓取中，跳过
- `first-start -> launching main.py` = 首次启动
- `resume -> launching main.py` = 断点续爬
- `cooling down, ~18 min left` = 冷却中，约还差 18 分钟
- `DONE: patagonia_output_xxx.xlsx ...` = 全部完成

**看抓取进度**：抓取中会弹出一个 Chrome 窗口；`output\patagonia_output_<任务号>_part.xlsx` 是滚动更新的部分结果，随时可打开看。

**停止**：右键运行 `windows-deploy\stop.ps1`，或：
```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\windows-deploy\stop.ps1
```
（会删除任务、结束抓取进程；断点保留，之后重新 install 可继续。）

---

## 五、完成后

- 全部抓完后，`output\` 下生成正式文件 `patagonia_output_YYYYMMDDHHMMSS.xlsx` + 同名 `_summary.txt`（含用时、商品数、SKU 数等汇总），`_part` 和断点被自动清理，旧结果归档到 `output\history\`。
- **监视任务会自动删除**：抓完后下一次检查监视会判"已完成"并**把自己这个计划任务从「任务计划」里删掉**（自动消失），不用手动清理。若想中途手动停，运行 `stop.ps1`/`stop.bat`。

**想再跑一次（刷新最新数据）**：确保 `output\` 里没有 `_part`/断点（完成后本就已清理），重新运行 `install_task.ps1` 即可（会重新全量抓取当天最新数据）。

---

## 六、注意事项（重要）

- **保持登录**：任务设为"仅在用户登录时运行"，因为爬虫要弹真实 Chrome 窗口。所以**这台电脑要保持登录状态**（可关显示器，但别注销/锁屏进省电睡眠）。建议在「电源选项」里把睡眠设为"从不"。
- **弹出的 Chrome 窗口别手动关**（抓取中它在工作）；被限流时程序会自己关掉它。
- **耗时**：直连（不用代理）会被 Akamai 按 IP 限流，约每抓 ~150 个商品就得停 30 分钟。640 个商品大约分 4~5 批、**总计约 4~5 小时**（大半是冷却等待）跑完。挂着过夜即可。
- **首次较慢**：第一次会建 `.chrome-profile\` 并热身。
- 若 PowerShell 提示脚本被禁止运行：用上面带 `-ExecutionPolicy Bypass` 的命令即可，无需改系统策略。
- **不需要管理员权限/授权**：install_task.ps1 做的都是普通用户权限内的事——
  - 移动/删除的是你自己 `output\` 目录里的文件；
  - `schtasks /Create` 建的是**当前用户自己的**任务（"仅登录时运行"），不用管理员、也不用输密码；
  - 用 `-ExecutionPolicy Bypass` 逐次绕过执行策略，不改系统设置。
  - 直接双击 .ps1 可能弹一次"是否运行脚本"的确认（这不是管理员/UAC，只是确认）；用上面的命令行方式跑就没有这个提示。
  - 只有想让任务"**不登录也运行**"（后台 session 0）时才需要管理员+存密码——但本方案故意不这么做（爬虫要弹真实 Chrome 窗口，需登录桌面），所以**全程不需要授权**。（例外：电脑被公司组策略锁死时另说。）
- 不要频繁重复跑；周期刷新建议间隔久一点（如每天/隔几天一次）。
