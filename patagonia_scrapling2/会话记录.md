## 一、初始需求

- 工程由 ChatGPT 生成，但**跑不起来**，需要修复。
- 核心问题：**没有解决 Patagonia 的反爬机制**，导致打不开网站。
- 目标：用 Python + Scrapling 爬取 `https://www.patagonia.jp/shop/womens`，
  输出 19 列数据到 `patagonia_output_YYYYMMDDHHMMSS.xlsx`，字段参考 `Patagonia-V8.2.xlsx`。
- 验证方式（用户后续给出）：
  1. 用参考 Excel 抽 10~20 个商品，比对各字段（库存/数量会变，其他应一致）；
  2. 最终跑 `/shop/womens`，抓 20~50 个商品。

## 二、排查过程（关键发现）

1. **首次运行**：`solve_cloudflare=True` 每次抛 `No Cloudflare challenge found`（该站没有
   Cloudflare 验证码）；且 Scrapling 响应正文在 `.body`/`get_all_text()`，代码却读 `.text`（恒空）。
2. **404 现象**：`/shop/*` 和 `/product/*` 一律返回 **10 字节 404，`Server: AkamaiNetStorage`**；
   而 `/`、`/home/`、`robots.txt`、`sitemap.xml` 正常 200。
3. **URL 未过期**：从当天更新的 `sitemap_0-product.xml` 取到存活商品 URL，照样 404 → 不是链接失效。
4. **不是 IP 问题**：本机 IP 为 **NTT / OCN 东京住宅宽带**（`hosting:false, proxy:false`）。
   加载 `/home/` 会拿到 `ak_bmsc`、`bm_sv` 等 Akamai cookie，但**始终拿不到关键的 `_abck`**。
5. **判定**：这是 **Akamai Bot Manager 硬拦截**，只放行成功运行传感器 JS、拿到有效 `_abck` 的客户端。
   - HTTP 伪装、Scrapling camoufox、Playwright **启动**的真实 Chrome（`navigator.webdriver=true`）全部被 404。

## 三、突破口

- 关键实验：**命令行启动的真实 Chrome**（`navigator.webdriver === false`），通过 CDP 挂载，
  先在 `/home/` 停留 + 滚动热身，之后直接 goto `/shop/womens/tops/t-shirts` → **200，230 个商品链接！**
- 商品页同样 200，含完整 `ProductGroup` JSON-LD 与 `product-schema`。
- 结论方案（全自动）：
  1. 命令行启动真实 Chrome（远程调试端口 + 独立 profile）→ `webdriver=false`；
  2. CDP 挂载并复用 `contexts[0]`（保留热身后的 Akamai cookie）；
  3. `/home/` 热身；之后 `/shop/*`、`/product/*` 全部 200。
- 注意：Scrapling 的 `DynamicSession` 走 CDP 会新建隔离上下文（无热身 cookie）→ 仍 404。
  因此改为**直接用 Playwright 驱动**（复用热身上下文），Scrapling 仅用作 HTML 解析器（`Selector`）。

## 四、代码改动

- `fetcher.py`：完全重写为 CDP 挂载 + 热身 + 复用上下文；返回带 `.status`/`.text` 的
  `FetchedPage`（内部包一个 Scrapling `Selector`）。新增自动启动 Chrome、自动发现 ws 端点。
- `scraper.py`：类别页改为浏览器滚动 + 点「さらに見る」加载全部商品；修复 `_read_url_file`
  把真实 URL 错误改写成 `/{model}.html`（导致 404）的 bug；404/正文检测改用 `get_all_text()`。
- `parser.py`：字段清洗对齐参考 —— `weight` 去掉 `(x oz)`；`material` 截到「お手入れ方法」前；
  `dimension` 重建为参考的干净表格；新增每色主图/图集处理（`{model}_{color}.jpg` + AltAssets 接口）。
- `cli.py`：新增 `--cdp-url/--chrome-path/--remote-port/--headless/--no-warmup/--concurrency`；
  去掉失效的 stealth 选项；类别 URL 改为从 `input_url.txt` 读取；输出目录默认 `output/`。
- 依赖新增 `playwright`；README/.gitignore/pyproject 同步更新。

## 五、字段验证结果

对照 `Patagonia-V8.2.xlsx`（多批次 15/20/50 商品，最多 752 个同 sku 行）：

| 字段 | 结果 |
|---|---|
| title, model, sku, brand, material, dimension, color, size | **100% 一致** |
| product_main_image, product_other_image | **一致**（每色主图 + AltAssets 图集，顺序对齐；仅个别商品网站数据本身错位） |
| msrp | 绝大多数一致，少数是**现价变动** |
| weight | 绝大多数一致，少数是**参考为空、本程序多抓到** |
| description | 取自页面 JSON-LD，跟随网站现文案（多为一个空格 / 保养文字的漂移） |
| stock_status, quantity | 实时数据，本就会变，不参与比对 |

**结论：无解析 bug，剩余差异全是网站数据随时间的正常变化，或本程序比参考更全。**

## 七、用户后续需求与处理

1. **并发**：新增 `--concurrency`（同热身上下文多标签共享 `_abck`），默认最终定为 **5**（建议 3–5）。
2. **输出目录**：默认改为自动创建的 `output/`，运行时无需指定。
3. **类别 URL 外置**：改为从 `input_url.txt` 读取（多行、`#` 注释、跨类别去重）。
4. **清理无用文件**：删除 13 个 `output_probe_*.xlsx`、`output_test.xlsx`、`__pycache__/`。
5. **README 中文化**：补充工程结构、运行方式、并发参数、部署到其他电脑的步骤。
6. **断点续爬 + 中途保存**（应对并发 5 时中途 not found、关窗口丢结果）：
   - 每爬完一个商品立即落盘到 `output/.checkpoint_<任务号>.jsonl`；重开时跳过已完成的，只爬剩余。
   - 每完成 5 个（`--flush-every`）写一次 `output/patagonia_output_<任务号>_part.xlsx` 部分结果。
   - 额外容错：抓到 404 时自动回 `/home/` 重新热身并重试一次（并发下加锁防重复热身）。
   - 全部完成才写正式时间戳文件并清理断点/`_part`；未完成退出码 1 并提示可续爬。新增 `checkpoint.py`。
7. **运行汇总 + 去重 + 汇总落盘**：结束打印并写 `_summary.txt`（含完成时间、用时、发现URL数、
   抓取商品数、去重后SKU数、去除重复数、状态）；写出前按 SKU 去重。
8. **历史文件归档**：多次运行时 `output/` 只留最新一份，历史结果自动移到 `output/history/`
   （`--no-archive` 可关）。任务号按爬取内容（类别URL/url-file）算，同输入不变、可续爬；
   最终文件是断点里"历次累积"的合并，完成后删 `_part`/断点。
9. **周期运行保证拿最新数据**：担心任务号不变导致续爬旧数据 → 加**断点时效**
   （`--resume-max-age-hours`，默认 24h）：断点超过 24h 自动丢弃、全新重抓。
   跑完即删断点 → 隔几天的周期运行永远全新；只有同一天崩溃重跑才续爬。`--no-resume` 可强制每次全新。
10. **被 Akamai 拦截（大量 404 / 发现 0 商品）的处理**：
    - 类别发现失败自动重试 3 次，中间**清 cookie + 重新热身**（`force_rewarm(clear_cookies=True)`）。
    - 发现 0 商品时不再输出混乱的"102/0"：判为 `discovery_failed`，保留断点、不写乱结果，退出码 2，
      并打印处置建议（等一会/`--fresh-profile`/降并发/手动验证）。
    - 部分类别被拦时不会误判"完成"（`failed_categories` 纳入完成判定，汇总里列出被拦类别）。
    - 新增 `--fresh-profile`（删 `.chrome-profile` 换全新身份，破解 cookie 中毒/卡死会话）。

## 八、最简用法

```powershell
python -m pip install -r requirements.txt   # 首次
python main.py                              # 读 input_url.txt，并发5，输出到 output\
python main.py --concurrency 3              # 想更稳时降并发
python main.py --max-products 50            # 先小批量试跑
```

需要 Google Chrome；首次运行会自动启动 Chrome、热身 Akamai 会话（较慢），之后复用
`.chrome-profile/` 更快。
